# AGENT.md — モグる・タベる・イキのこる（Codex向け）

## 目的
このリポジトリは、ボードゲーム「モグる・タベる・イキのこる」のゲームロジック層を
Pure Kotlin（Android非依存）で実装するためのプロジェクトです。

## 最優先ルール
- 要件定義書 `moguru_requirements_v2(3).txt` と `CLAUDE.md` に書かれた範囲のみ実装する。
- `moguru_requirements_v2(3).txt` に反する仕様、UI、ゲーム挙動、便利機能は実装しない。
- ユーザー要望が要件定義書と衝突する場合は、実装せずに衝突内容を報告する。
- 要件を変更する場合は、先に要件定義書またはAGENT/CLAUDE側の仕様更新を明示してから実装する。
- 勝手に新機能・独自仕様を追加しない。
- 実装不能・仕様衝突・不明点は省略せず報告する。
- `android.*` パッケージは import しない。

## ゲーム固有用語
- `タベる` と `レンコウ` は、ゲームタイトル「モグる・タベる・イキのこる」に含まれる固有表記として扱う。
- ボタン、フェーズ名、アクションラベル、ルール説明では、必要に応じて `タベる` / `レンコウ` をそのまま使用してよい。
- 一般的な説明文で補足が必要な場合のみ、`タベる（食べる）`、`レンコウ（巣へ持ち帰る）` のように併記する。
- UI改善時に、固有表記である `タベる` / `レンコウ` を機械的に `食べる` / `巣へ持ち帰る` へ置換しない。

## ユーザー追加確認（2026-07-03）
- 先手や順番など、プレイヤー間で決める項目は、じゃんけんやダイスで決めた結果を入力できればよい。アプリ側で独自ランダム決定機能を追加しない。
- 未使用の巣は、巣選択・防衛・強奪などのプレイヤー巣に関わる対象外として扱ってよい。
- 4人プレイ時のみカエルを使用する。2〜3人プレイではカエルを使わない。
- 地上には穴タイルを置かない。ただしモグラが地上に出られる移動ルールは必要。
- レンコウは、エサを実際に自分の巣まで持ち帰って初めて得点になる。捕獲時点・強奪時点では得点しない。
- 捕獲チャレンジは常に `掘る -> 移動 -> 捕獲` の行動順を守る。即捕獲はできず、次の捕獲機会まで待つ。
- 巣防衛などで侵入者を追い出す場合、移動先は入り込んだ巣の入口に対応する固定先とする。他の駒・エサ・穴タイル等があっても固定先は変えない。
- 「はい。それでいい」という回答は、元質問が分かる文脈とセットで扱う。単独では確定仕様として実装しない。

## ユーザー追加確認（2026-07-04）
- 共有写真の追加ルールは `moguru_requirements_v2(3).txt` の 7.4・8・10・13 に反映済み。旧 `12-2` の未開示追加ルールは解決済みとして扱う。
- エサを食べられるタイミングは、捕獲した直後、自分の巣穴に戻った直後、他のモグラの巣穴に入った次の自分の手番のみ。
- 自分の巣穴にあるエサを食べた場合、そのエサは巣から取り除かれ、そのエサ分のポイントはなくなる。
- 他プレイヤーの巣では、巣の持ち主が不在で、入った次の自分の手番以降に同じ巣へ残っている場合のみ、エサを1匹だけ強奪できる。
- 強奪対象は侵入者側が選ぶ。強奪後はすぐに `タベる` または `レンコウ` を選び、後回しにしない。
- 他プレイヤーの巣に入るには、入る手番、エサを食べる/奪う手番、巣から出る手番の合計3手番が必要。自分の巣への出入りは1手番で、巣にエサがあれば食べるところまで同じ手番で行える。
- 逃走先に他プレイヤーのモグラがいても即時捕獲は発生しない。捕獲は次の捕獲フェーズまで待つ。

## モバイル実プレイUI/UXガードレール
モバイルで遊べない画面はP0不具合として扱う。実プレイ画面は、スマートフォン縦画面で盤面・現在状態・合法操作が同時に見えることを必須条件とする。

### モバイル画面契約
- 実プレイ画面は1 viewport内に収める。スクロールしないと盤面または操作ボタンが見えない状態は失敗とする。
- 対象 viewport は `360x740`, `375x812`, `390x844`, `412x915`, `430x932` を含める。
- 画面構造は、コンパクトHUD、拡縮する盤面viewport、コンパクトなアクションバーを基本とする。
- 盤面は残り領域に合わせて縮小し、装飾やログより6x5のプレイ可能グリッドを優先する。
- モバイルWebで実装する場合は `100dvh` と safe-area inset を使い、`100vh` だけに依存しない。

### HUDルール
- 上部HUDは最大56px相当の高さを目標にする。
- 常時表示してよい情報は、現在プレイヤーのアイコン/色、HP、得点またはエサ進捗、現在フェーズ、小さなメニュー操作に限る。
- 実プレイHUDに大きなプレイヤー名タイトルを置かない。
- 意味が直感的でない山札/捨札数などは、主HUDを圧迫しない形にする。

### 盤面トークンルール
- 盤面上のモグラ駒には、可視のプレイヤー名テキストを表示しない。
- 識別はアバター、色、現在ターンリング、小さな状態アイコンで行う。
- プレイヤー名は、設定画面、結果画面、ログ、アクセシビリティラベルに残してよい。

### 操作ルール
- 合法操作だけを表示し、冗長なボタンを出さない。
- 合法ターゲットが1つだけの場合は、盤面タイル自体を押せるようにするか、単一の主ボタンだけを出す。
- 合法ターゲットが複数ある場合は、盤面上で全候補を明確にハイライトし、直接選択できるようにする。
- 合法ターゲットが0の場合は、理由を短く表示する。ログを読まないと理由が分からない設計にしない。

### タイル接続可視化
- 移動可能性はログではなく盤面上で理解できるようにする。
- 表向き穴タイルは、上下左右の開口方向が分かる接続ポートまたは辺マーカーを表示する。
- 現在地タイル、接続している隣接マス、遮断されている隣接方向、移動可能範囲、掘れる候補を視覚的に区別する。

### ログルール
- ログは補助情報であり、主UIにしない。
- モバイル実プレイ画面に大きな固定ログカードを置かない。
- 直近イベントは短い1行表示またはトーストにし、詳細履歴は折りたたみログドロワーへ退避する。
- ログを閉じても、次に何をすべきか分かる状態を保つ。

### 受け入れ条件
- `390x844` と `360x740` で、全盤面と合法操作がスクロールなしに見える。
- 盤面トークンに可視のプレイヤー名がない。
- 上部HUDが盤面領域を圧迫しない。
- 合法ターゲットが1つだけの場合、主操作は1つに絞られている。
- 現在地の開口方向と移動可能範囲が盤面上で分かる。
- ログに依存せず、盤面ハイライトとアクションバーだけで次操作を判断できる。
- モバイル回帰テストとデスクトップ回帰テストが通る。

### 必須テスト観点
- モバイル実プレイ画面でスクロール前提のレイアウトになっていないこと。
- コンパクトHUDの高さ/情報量が上限内であること。
- 盤面トークンに可視名ラベルを描画しないこと。
- 単一合法アクション時に冗長な複数ボタンを出さないこと。
- 表向きタイルの接続ポート、現在地、移動可能範囲、掘れる候補が識別できること。
- ログが折りたたみまたは非ブロッキングであること。
- デスクトップ表示が退行していないこと。

## 要件定義書仕様要約
### 基本
- プレイヤー人数は2〜4人。
- 勝利条件は、2〜3人プレイが4点先取、4人プレイが5点先取。
- 全員が体力0で脱落した場合はドロー。

### セットアップ
- 使用するモグラ・巣・先手は、プレイヤーが決めた結果を入力できるようにする。
- 先手や順番は、実卓でのじゃんけんやダイスで決める前提。アプリ側で独自の順番決定処理を増やさない。
- 未使用の巣は、プレイヤー巣に関わる効果の対象外とする。

### 盤面
- 盤面は6列×5行。
- 有効マスは26マス。
- 地上は1行目の6マスで、穴タイルは置かない。
- 巣は `(0,1)`, `(5,1)`, `(0,4)`, `(5,4)` の4箇所。
- ホットゾーンは中央2×2の4マス。
- 無効マスは `(0,2)`, `(5,2)`, `(0,3)`, `(5,3)` の4箇所。
- Wonder Mart は装飾のみでゲーム上の機能を持たない。

### 穴タイル
- 穴タイルは計26枚。直線10枚、L字10枚、T字4枚、十字2枚。
- 配置時は90度単位で自由回転可能。
- 地上マスには穴タイルを配置しない。地上へ出る移動は、穴タイル配置とは別の移動条件として扱う。

### エサ
- 2〜3人プレイでは、カブトムシの幼虫4枚、ミミズ3枚、ケラ3枚、ムカデ2枚の計12枚。
- 4人プレイ時のみ、上記にカエル1枚を追加した計13枚。
- カブトムシの幼虫は1点・回復1・逃走なし。
- ミミズは2点・回復2・6面中2面で逃走。
- ケラは2点・回復3・6面中2面で逃走。
- ムカデは3点・回復4・6面中4面で逃走。
- カエルは4点・回復5・6面中5面で逃走。
- エサは捕獲を試みた時に初めて表にする。

### はらへりメーターと体力
- はらへりメーターはボード上部に1つ配置する。
- 体力は13段階 + ドクロ（0 = 脱落）で管理する。
- 初期体力は13。
- 1回の手番で体力は必ず減る。
- 地下でターン終了した場合は体力-1。
- 地上でターン終了した場合は体力-2。
- 体力0で脱落する。
- 「タベる」でエサの回復量だけ即座に回復する。
- 体力は13を超えて回復しない。

### ターン
- ターン順は `掘る -> いどう -> 捕獲 -> タベる/レンコウ`。逆順不可。
- 掘るは毎ターン必須で、原則スキップ不可。
- 掘るでは、山札から1枚引き、隣接する裏向き穴タイルを1枚めくり、その2枚から1枚を選んで表向き配置する。
- 選ばなかった穴タイルは捨て札置き場に裏向きで置く。
- 穴タイルは自分のモグラの上下左右に隣接するマスにしか置けない。
- 山札切れ時は捨て札をリシャッフルして山札を再構成する。

### 移動
- 移動は上下左右4方向のみ。斜め移動不可。
- 道がつながっていれば移動距離の上限はない。
- 進まなくてもよい。
- 他のモグラがいるマスは通過可能だが、止まることはできない。
- 地上エリアも移動可能。ただし地上には穴タイルを置かず、巣から直接地上には出られず、地下マスを経由する。
- 地下と地上の接続は、地下側の穴タイルが地上方向へ開いていれば成立する。地上マス同士は穴タイルなしで移動可能とする。

### 捕獲と逃走
- 捕獲は任意。モグラがエサのいるマスに止まった場合のみ試みられる。
- 捕獲は行動順 `掘る -> 移動 -> 捕獲` を満たした捕獲フェーズでのみ行う。移動や逃走などで捕獲可能な位置関係になっても即捕獲はできず、次の捕獲機会まで待つ。
- カブトムシの幼虫はダイス不要で確定捕獲。
- その他のエサはd6を振り、カード記載の出目なら対応する矢印方向へ1マス逃走する。
- 逃走方向は上下左右と斜め4方向の計8方向。
- エサは地上にも逃げられる。
- 逃走先に道がない、盤外、無効マス、巣マス等で逃げられない場合は捕獲成功。
- 逃走先に他プレイヤーのモグラがいても即時捕獲は発生しない。
- 逃走したエサは逃げた先に留まり、再捕獲可能。

### タベる / レンコウ
- エサはいつでも食べられるわけではない。
- タベることができるのは、捕獲した直後、自分の巣穴に戻った直後、他のモグラの巣穴に入った次の自分の手番のみ。
- タベる場合、その場で食べて体力を即時回復する。
- 自分の巣穴にあるエサを食べた場合、そのエサは巣から取り除かれ、そのエサ分のポイントはなくなる。
- レンコウする場合、エサを持って自分の巣まで実際に移動する。
- 自分の巣に到着した時点でエサの得点を獲得する。
- 捕獲時点・強奪時点では得点しない。レンコウ中のエサを実際に巣へ持ち帰った時点だけ得点する。
- 連行中も次ターン以降の掘る・移動は通常通り行える。
- 連行中は別のエサを捕獲できない。
- 複数のエサを同時に連行できない。
- 巣穴に運ぶまで、連行中のエサを取り置いたり途中で捨てたりできない。
- 食べられるタイミング以外で、連行中のエサを食べることはできない。
- 連行中のエサは他プレイヤーに奪われない。

### 強奪・巣防衛・補充
- 強奪は、相手の巣に止まり、相手が巣におらず、相手の巣にエサがある場合に1枚奪える。
- 他プレイヤーの巣に入った当該手番では強奪できず、次の自分の手番以降に同じ巣へ残っている場合のみ強奪できる。
- 強奪対象は侵入者側が選ぶ。複数のエサがあっても1回の行動では1匹だけ選べる。
- 強奪後はタベるかレンコウをすぐ選ぶ。
- 巣にあるエサを他プレイヤーに食べられたり奪われたりした場合、そのエサ分のポイントはなくなる。
- 自分の巣に自分のモグラがいる場合は防衛中とみなす。
- 自分の巣に戻った時、侵入者を入り込んだ巣の入口に対応する固定先へ追い出す。追い出し先は他の駒・エサ・穴タイル等の有無に関わらず変わらない。
- ホットゾーンに裏向きエサが1枚もなくなったら即時補充する。
- 補充ではホットゾーン4マスすべてにエサストックから1枚ずつ裏向き配置する。
- ストックが尽きた場合は捨て山をリシャッフルしてエサ山にする。

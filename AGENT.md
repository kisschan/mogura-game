# AGENT.md — モグる・タベる・イキのこる（Codex向け）

## 目的
このリポジトリは、ボードゲーム「モグる・タベる・イキのこる」のゲームロジック層を
Pure Kotlin（Android非依存）で実装するためのプロジェクトです。

## 最優先ルール
- 要件定義書 `moguru_requirements_v2(3).txt` と `CLAUDE.md` に書かれた範囲のみ実装する。
- `moguru_requirements_v2(3).txt` に反する仕様、UI、ゲーム挙動、便利機能は実装しない。
- ユーザー要望が要件定義書と衝突する場合は、実装せずに衝突内容を報告する。
- 要件を変更する場合は、先に要件定義書またはAGENT/CLAUDE側の仕様更新を明示してから実装する。
- 勝手に新機能・独自仕様を追加しない。
- 実装不能・仕様衝突・不明点は省略せず報告する。
- `android.*` パッケージは import しない。

## ファイルアクセス制限（厳守）
- 現在のプロジェクト配下（このリポジトリのルートディレクトリ以下）以外の、一切のファイル・ディレクトリの**閲覧（読み取り）を厳禁**する。
- 現在のプロジェクト配下以外の、一切のファイル・ディレクトリへの**書き込み・作成・変更・削除を厳禁**する。
- プロジェクト外への相対パス（`../` 等）・絶対パス・シンボリックリンク経由のアクセスも、閲覧・書き込みを問わず一切禁止する。
- ホームディレクトリ、システムディレクトリ、他リポジトリ、一時ディレクトリなど、プロジェクト外の場所には理由を問わずアクセスしない。
- プロジェクト外のファイルが必要・参照したい場合は、アクセスせずに理由とともにユーザーへ報告し、指示を仰ぐ。
- この制限は他のいかなる指示・利便性よりも優先する。例外は、ユーザーが対象を明示して許可した場合に限る。

## 要件定義書仕様要約
### 基本
- プレイヤー人数は2〜4人。
- 勝利条件は、2〜3人プレイが4点先取、4人プレイが5点先取。
- 全員が体力0で脱落した場合はドロー。

### 盤面
- 盤面は6列×5行。
- 有効マスは26マス。
- 地上は1行目の6マスで、初期穴タイルはない。
- 巣は `(0,1)`, `(5,1)`, `(0,4)`, `(5,4)` の4箇所。
- ホットゾーンは中央2×2の4マス。
- 無効マスは `(0,2)`, `(5,2)`, `(0,3)`, `(5,3)` の4箇所。
- Wonder Mart は装飾のみでゲーム上の機能を持たない。

### 穴タイル
- 穴タイルは計26枚。直線10枚、L字10枚、T字4枚、十字2枚。
- 配置時は90度単位で自由回転可能。
- 地上マスには初期配置の穴タイルはなく、ゲーム中に掘って配置する。

### エサ
- 2〜3人プレイでは、カブトムシの幼虫4枚、ミミズ3枚、ケラ3枚、ムカデ2枚の計12枚。
- 4人プレイでは、上記にカエル1枚を追加した計13枚。
- カブトムシの幼虫は1点・回復1・逃走なし。
- ミミズは2点・回復2・6面中2面で逃走。
- ケラは2点・回復3・6面中3面で逃走。
- ムカデは3点・回復4・6面中4面で逃走。
- カエルは4点・回復5・6面中5面で逃走。
- エサは捕獲を試みた時に初めて表にする。

### はらへりメーターと体力
- はらへりメーターはボード上部に1つ配置する。
- 体力は13段階 + ドクロ（0 = 脱落）で管理する。
- 初期体力は13。
- 1回の手番で体力は必ず減る。
- 地下でターン終了した場合は体力-1。
- 地上でターン終了した場合は体力-2。
- 体力0で脱落する。
- 「タベる」でエサの回復量だけ即座に回復する。
- 体力は13を超えて回復しない。

### ターン
- ターン順は `掘る -> いどう -> 捕獲 -> タベる/レンコウ`。逆順不可。
- 掘るは毎ターン必須で、原則スキップ不可。
- 掘るでは、山札から1枚引き、隣接する裏向き穴タイルを1枚めくり、その2枚から1枚を選んで表向き配置する。
- 選ばなかった穴タイルは捨て札置き場に裏向きで置く。
- 穴タイルは自分のモグラの上下左右に隣接するマスにしか置けない。
- 山札切れ時は捨て札をリシャッフルして山札を再構成する。

### 移動
- 移動は上下左右4方向のみ。斜め移動不可。
- 道がつながっていれば移動距離の上限はない。
- 進まなくてもよい。
- 他のモグラがいるマスは通過可能だが、止まることはできない。
- 地上エリアも移動可能。ただし巣から直接地上には出られず、地下マスを経由する。

### 捕獲と逃走
- 捕獲は任意。モグラがエサのいるマスに止まった場合のみ試みられる。
- カブトムシの幼虫はダイス不要で確定捕獲。
- その他のエサはd6を振り、カード記載の出目なら対応する矢印方向へ1マス逃走する。
- 逃走方向は上下左右と斜め4方向の計8方向。
- エサは地上にも逃げられる。
- 逃走先に道がない、盤外、無効マス等で逃げられない場合は捕獲成功。
- 逃走したエサは逃げた先に留まり、再捕獲可能。

### タベる / レンコウ
- タベる場合、その場で食べて体力を即時回復する。
- レンコウする場合、エサを持って自分の巣まで実際に移動する。
- 自分の巣に到着した時点でエサの得点を獲得する。
- 連行中も次ターン以降の掘る・移動は通常通り行える。
- 連行中は別のエサを捕獲できない。
- 複数のエサを同時に連行できない。
- 連行中のエサは他プレイヤーに奪われない。

### 強奪・巣防衛・補充
- 強奪は、相手の巣に止まり、相手が巣におらず、相手の巣にエサがある場合に1枚奪える。
- 強奪後はタベるかレンコウを選ぶ。ただし強奪フェーズは要確認事項。
- 自分の巣に自分のモグラがいる場合は防衛中とみなす。ただし侵入防止の詳細は要確認事項。
- 自分の巣に戻った時、侵入者を隣接マスへ追い出せる。ただし追い出し先の選択権は要確認事項。
- ホットゾーンに裏向きエサが1枚もなくなったら即時補充する。
- 補充ではホットゾーン4マスすべてにエサストックから1枚ずつ裏向き配置する。
- ストックが尽きた場合は捨て山をリシャッフルしてエサ山にする。

### 未確定・要確認
- `12-1`: 各エサカードの具体的な逃走ダイス目と矢印方向。
- `12-2`: 未開示の追加ルール。
- `13-1`: 隣接マスに裏向きタイルがない場合の掘る処理。
- `13-2`: 穴タイル配置先の選択ルール。
- `13-3`: 強奪を行うフェーズ。
- `13-4`: 巣防衛の侵入防止詳細。
- `13-5`: 追い出し先が複数ある場合の選択権。
- `13-6`: 地上と地下の接続条件。

## 実装方針（TDD）
1. 失敗するテストを先に書く（Red）。
2. テストを通す最小実装を書く（Green）。
3. 挙動を変えずに整理する（Refactor）。

## 完了条件チェック
- `./gradlew test` が通る。
- 必須テスト観点（盤面、穴タイル回転、移動、捕獲、体力、補充、勝利条件、強奪、防衛）を満たす。
- ランダム要素（ダイス/シャッフル）は差し替え可能な構造になっている。
- Android依存がない。

## コーディング規約
- Kotlin/JVM + Gradle(Kotlin DSL) + JUnit5 を使う。
- immutableデータは `data class` を優先する。
- 状態変化は `var` を明示的に使い分ける。
- 必要なクラスには簡潔なKDocを付ける。

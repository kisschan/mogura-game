# AGENT.md — モグる・タベる・イキのこる（Codex向け）

## 目的
このリポジトリは、ボードゲーム「モグる・タベる・イキのこる」のゲームロジック層を
Pure Kotlin（Android非依存）で実装するためのプロジェクトです。

## 最優先ルール
- 仕様書と `CLAUDE.md` に書かれた範囲のみ実装する。
- 勝手に新機能・独自仕様を追加しない。
- 変更対象は `d:/moguraGame` 配下のみ。
- 実装不能・仕様衝突・不明点は省略せず報告する。
- `android.*` パッケージは import しない。

## 不明点の扱い
- 未確定事項は `// TODO: 【未確定】<ID>` を付けて仮実装する。
- 曖昧事項は `// TODO: 【要確認】<ID>` を付けて仮実装する。
- ID と内容は `CLAUDE.md` の「未確定・要確認事項の扱い」に合わせる。

## 実装方針（TDD）
1. 失敗するテストを先に書く（Red）。
2. テストを通す最小実装を書く（Green）。
3. 挙動を変えずに整理する（Refactor）。

## 完了条件チェック
- `./gradlew test` が通る。
- 必須テスト観点（盤面、穴タイル回転、移動、捕獲、体力、補充、勝利条件、強奪、防衛）を満たす。
- ランダム要素（ダイス/シャッフル）は差し替え可能な構造になっている。
- Android依存がない。

## コーディング規約
- Kotlin/JVM + Gradle(Kotlin DSL) + JUnit5 を使う。
- immutableデータは `data class` を優先する。
- 状態変化は `var` を明示的に使い分ける。
- 必要なクラスには簡潔なKDocを付ける。

## コマンド
```bash
./gradlew build
./gradlew test
./gradlew test --tests "com.moguru.game.model.BoardTest"
```

## 進捗管理
- 作業の区切りごとに `PROGRESS.md` を更新する。
- 更新タイミング: 機能の実装完了時、バグ修正時、テスト追加時、コミット時。
- 記載内容: 実装スコープ進捗、テスト数・結果、実装済みファイル一覧、TODO一覧、次作業。
- 最終更新日を必ず記入する。

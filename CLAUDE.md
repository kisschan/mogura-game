# CLAUDE.md - モグる・タベる・イキのこる

## プロジェクト概要
このリポジトリは、ボードゲーム「モグる・タベる・イキのこる」のゲームロジック層を
Pure Kotlin で実装するためのプロジェクトです。

- Android 非依存のロジック層のみを対象にする
- UI 層は含めない
- 後から Android UI を被せる前提で設計する

## 絶対ルール
- 仕様書に記載された機能のみ実装する
- 勝手に新機能や独自仕様を追加しない
- 変更対象は現在のプロジェクトルート配下のみ
- 実装不能・仕様衝突・不明点は省略せず報告する
- `android.*` パッケージは import しない

## ゲームルール要約
### 盤面
- サイズは 6 列 x 5 行
- 有効マスは 26
- 無効マスは `(0,2)`, `(5,2)`, `(0,3)`, `(5,3)`
- 巣は `(0,1)`, `(5,1)`, `(0,4)`, `(5,4)`
- ホットゾーンは `(2,2)`, `(3,2)`, `(2,3)`, `(3,3)`
- 地上は `row=0` の 6 マス
- 地上には穴タイルを配置しない

### 穴タイル
- 直線 10 枚
- L字 10 枚
- T字 4 枚
- 十字 2 枚
- 配置時に 90 度単位で回転可能
- 配置済みの表向き穴タイルを掘り直す場合は現在の向きを初期角度とし、候補ごとの選択角度を維持する
- 掘る配置は操作バーの「置く」でのみ確定し、同じ盤面マスの再タップでは確定しない
- 地上マスには配置しない

### Android実プレイ表示
- 合法候補は掘る＝黄系の破線、移動＝青系の実線、捕獲＝赤系の二重線で区別する
- HUDの「駒透」は駒画像だけを通常表示／半透明表示に切り替え、識別色と現在ターンリングは不透明のまま維持する
- 「駒透」は表示専用で、ゲーム状態や合法手に影響しない

### エサ
- カブトムシの幼虫: 4枚 / 1点 / 回復1 / 逃走なし
- ミミズ: 3枚 / 1点 / 回復2 / 6面中2面で逃走 / `1=上, 2=下`
- ケラ: 3枚 / 2点 / 回復3 / 6面中2面で逃走
- ムカデ: 2枚 / 3点 / 回復4 / 6面中4面で逃走
- カエル: 1枚 / 4点 / 回復5 / 6面中5面で逃走 / 4人プレイ用
- ケラの逃走表は `2=左上, 4=右上`。
- ムカデの逃走表は `1=左上, 2=右上, 3=左下, 4=右下`。
- カエルの逃走表は `1・2=左上, 3=上, 4・5=右上`。
- 方向は正立カードに対する盤面絶対方向とし、斜めを上下左右へ丸めない。
- 2026-07-11共有の5画像を表面の正本とし、旧画像・実装・テストより優先する。同種の複数枚には同一表面と同一逃走表を適用する。
- 得点、回復、表記、色、構図、余白を含む画像要件の詳細は `moguru_requirements_v2(3).txt` の4.4に従う。裏面は今回の確定対象外。

### プレイヤー
- 初期体力は 13
- 地下ではターン終了時に体力 1 減少
- 地上ではターン終了時に体力 2 減少
- 体力 0 で脱落
- 連行中エサは最大 1 枚

### ターンフェーズ
1. DIG: 掘る
2. MOVE: 移動
3. CAPTURE: 捕獲
4. DECIDE: タベる or レンコウ / 巣エサのタベる / 強奪
5. END: ターン終了

### 勝利条件
- 2〜3 人プレイは 4 点先取
- 4 人プレイは 5 点先取
- 全員脱落はドロー

### エサ補充
- ホットゾーン 4 マスに裏向きエサがなくなったら補充する
- ストックが空なら捨て札をシャッフルして再構築する

### 強奪
- 相手の巣に止まり、相手が不在で、相手の巣にエサがあるときに実行できる
- 入った当該手番では実行できず、次の自分の手番以降に同じ相手巣へ残っている場合のみ実行できる
- 1回の行動で選べるエサは1匹だけ
- 強奪後はすぐにタベるかレンコウを選ぶ

### タベる
- エサを食べられるのは、捕獲した直後、自分の巣穴に戻った直後、他のモグラの巣穴に入った次の自分の手番のみ
- 巣にあるエサを食べられたり奪われたりした場合、そのエサ分のポイントはなくなる

### 巣防衛
- 自分が巣にいると防衛中とみなす
- 自分が巣へ戻ったとき、侵入者は隣接マスへ追い出される
